# Ralph Loop 평가 기준

Kevin 피드백 + Jaden 피드백 기반 리포트 품질 체크리스트.

## 1. 첫 페이지 임팩트 (Shock Value)
- [ ] 총 코드 변경량 (additions + deletions) 큰 숫자로 표시
- [ ] 활성 레포지토리 수 (전체)
- [ ] 기여자 수
- [ ] 신문/잡지 1면처럼 시선을 잡는 구성
- [ ] "투자자가 추가 투자하고 싶게" 만드는 톤

## 2. 전체 레포 포함
- [ ] 모든 활성 레포가 포함됨 (상위 10개가 아닌 전체)
- [ ] 각 레포별 코드 추가/삭제 수치
- [ ] 양으로 밀기 — 수십~수백 페이지 OK

## 3. 콘텐츠 깊이
- [ ] 레포별 목표/방향 설명
- [ ] 커밋 기반 구체적 작업 내용
- [ ] 코드 추가/삭제에 대한 맥락 (왜 이 변경이 의미있는지)
- [ ] 기여자 정보 포함

## 4. ATI 페이지 대비
- [ ] ATI Code Stats 페이지보다 더 많은 정보
- [ ] 숫자 + 내러티브 결합 (ATI는 숫자만)

## 5. 톤 & 포맷
- [ ] 공격적/자신감 있는 톤 ("더 투자해라" 느낌)
- [ ] 방어적이지 않음 ("아직 활동 중" 아님)
- [ ] Medium 게시 적합 포맷 (마크다운)
- [ ] 독자 절반이 AI임을 고려한 구조화

## 점수 기준
각 항목 0-2점 (0=미달, 1=부분충족, 2=충족)
합산 30점 만점 → 24점(80%) 이상이면 PASS
